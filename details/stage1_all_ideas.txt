# P1003 铺地毯 - 解题思路

## 第一步：理解问题本质

### 问题的核心是什么？
- 每张地毯是一个矩形区域，由左下角坐标(a,b)和长度g、k决定
- 覆盖区域为：**x ∈ [a, a+g]**，**y ∈ [b, b+k]**（边界也算覆盖）
- 关键点：**后铺的地毯覆盖在上面**，所以我们需要找的是**最上面**的那张，即**编号最大的**覆盖该点的地毯

### 最简单实例
- n = 0：没有地毯，输出 -1
- n = 1：只有一张地毯，直接判断点是否在矩形内

## 第二步：确定算法策略

### 分析问题特征
- 这是一个**查询问题**：给定一个点，找覆盖它的最上面的地毯
- 数据规模：n ≤ 10^4，这个规模O(n)的算法完全可以接受
- 核心操作：判断一个点是否在矩形内

### 思路
由于后铺的地毯覆盖上面的，我们只需要**从后往前**遍历（从n到1），找到第一个覆盖该点的地毯即可。

### 判断点是否在矩形内
对于地毯i (a, b, g, k)，点(x, y)被覆盖当且仅当：
- a ≤ x ≤ a+g（注意是≤，因为边界也算）
- b ≤ y ≤ b+k

## 第三步：算法步骤

1. 读取n
2. 读取n张地毯的信息，存入数组
3. 读取查询点(x, y)
4. **从后往前遍历**地毯（i = n 到 1）：
   - 判断点(x, y)是否在当前地毯的覆盖范围内
   - 如果是，输出当前地毯编号，结束
5. 如果遍历完所有地毯都没找到，输出-1

## 第四步：复杂度分析

- **时间复杂度**：O(n)，只需遍历一次地毯
- **空间复杂度**：O(n)，需要存储所有地毯信息（也可以边读边处理，进一步优化到O(1)）

## 总结

这是一个简单的**顺序查找**问题，关键在于：
1. 理解题目中"后铺的覆盖在上面"意味着从后往前找
2. 注意边界条件（边界也算覆盖，所以用≤而非<）
3. 找到第一个匹配的地毯后立即输出，因为它是编号最大的，也就是最上面的

成员 3: ### 第一步：理解问题，从最简单的情形入手

#### a. 构造最简单实例
- **最简单实例**：`n = 0`，即没有地毯。此时无论查询点是什么，结果都应为 `-1`。
- 次简单实例：`n = 1`，只有一张地毯，比如输入为：
  ```
  1
  0 0 2 2
  1 1
  ```
  表示在左下角 (0,0) 铺一张 2×2 的地毯，查询点 (1,1) 显然在其中，输出应为 `1`。

#### b. 求解与验证最简单实例
- 对于 `n=0`：无地毯，任何点都不被覆盖 → 输出 `-1`，正确。
- 对于 `n=1`：判断点是否在该矩形内（含边界）。若在，则输出编号 1；否则输出 -1。
- 手动验证可知：点 (x,y) 是否落在 `[a, a+g] × [b, b+k]` 范围内即可判定是否被覆盖。

#### c. 枚举所有可行解
- 本题不是最优化问题，而是**查询定位问题**：给定一个点，找覆盖它的**最上面一张地毯**。
- 所有可行解就是所有覆盖该点的地毯编号，我们要的是其中**编号最大**的那个（因为后铺的在上面）。

### 第二步：联想类似问题

#### a. 唤醒知识储备
- 与“**矩形覆盖**”、“**点在矩形内判定**”相关。
- 类似“**区间覆盖**”的二维扩展。
- 每个地毯是一个轴对齐矩形，操作是按顺序叠加。
- 查询问题是：给定点，找最后覆盖它的矩形。

#### b. 尝试直接应用
- 若使用类似“扫描线”或“线段树”处理矩形覆盖，属于高级算法，适用于大规模更新和查询。
- 但本题中：
  - 地毯数量 `n ≤ 10^4`
  - 查询点仅一个
  - 所有地毯静态（不修改）
- 因此无需复杂数据结构。

#### c. 改进与适配
- 直接思路：从最后铺设的地毯（编号最大）开始，往前逆序检查每张地毯是否覆盖查询点。
- 一旦找到第一个能覆盖该点的地毯，就是最上面的那张 → 返回其编号。
- 若遍历完都没有，则返回 `-1`。
- 时间复杂度：O(n)，可接受（`n ≤ 10^4`）。

### 第三步：复杂实例与分解

#### a. 构造复杂实例
例如：
```
4
0 0 3 3    // 编号1
1 1 2 2    // 编号2
2 2 2 2    // 编号3
1 3 2 1    // 编号4
2 2
```
查询点 (2,2)：
- 被编号1覆盖？是（[0,3]×[0,3]）
- 被编号2覆盖？是（[1,3]×[1,3]）
- 被编号3覆盖？是（[2,4]×[2,4]）
- 被编号4覆盖？否（y 范围 [3,4]，2 不在）
- 最晚铺设的是编号3 → 输出 `3`

#### b. 分析可分解性
- 输入是地毯列表，可看作一个数组，可以按顺序或逆序遍历。
- 每个地毯独立，覆盖判断互不影响，**问题具有可分解性**：每个地毯单独判断是否覆盖查询点。
- 但由于覆盖顺序影响结果（后铺的优先），不能并行决定答案。

#### c. 分析可组合性
- 输出只是一个整数：最大编号的覆盖地毯。
- 可以将整个问题拆解为：对每个地毯 i，判断是否覆盖 (x,y)，记录满足条件的最大 i。
- 因此，**解具有可组合性**：逐个判断，保留最大编号即可。

---

### 第四步：确定解题策略

#### 选择策略：**逆序遍历 + 几何判断**

- **核心观察**：后铺设的地毯在上方，因此我们应从编号最大的地毯（第 n 张）开始，向前逐个判断是否覆盖查询点。
- 一旦发现某张地毯覆盖了该点，它就是最上面的一张 → 直接返回其编号。
- 如果从后往前都没有覆盖，则返回 `-1`。

- **关键操作**：判断点 (x, y) 是否位于矩形 [a, a+g] × [b, b+k] 内（闭区间）。
  - 即：`a <= x <= a+g` 且 `b <= y <= b+k`

- **算法策略**：不需要贪心、动态规划、分治或回溯。
- **本质**：**模拟 + 逆序查找**

- **时间复杂度**：O(n)，最优（必须至少读入所有地毯）
- **空间复杂度**：O(n)，用于存储地毯信息（也可优化为边读边存，查询时逆序访问）

---

### 总结思路：

1. 读取 n，存储每张地毯的 (a, b, g, k)，即左下角和尺寸。
2. 读取查询点 (x, y)。
3. 从编号最大的地毯（第 n 张）开始，逆序遍历到第 1 张：
   - 计算当前地毯的右上角坐标：`(a + g, b + k)`
   - 判断 (x, y) 是否落在 `[a, a+g]` 和 `[b, b+k]` 的闭区间内
   - 如果是，输出当前地毯编号，结束程序
4. 如果遍历完都没有覆盖，输出 `-1`

> 该方法充分利用铺设顺序决定了覆盖顺序的特点，通过**逆序查找第一个命中者**来高效求解。
