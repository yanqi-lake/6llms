#### 第一步：理解问题，从最简单的情形入手

1. **构造最简单实例**：
   - 最简单的情况是 `n = 1`，即只有一张地毯。
   - 输入：`1\n0 0 2 2\n1 1`
   - 输出：`1`

2. **求解与验证最简单实例**：
   - 手动计算点 `(1, 1)` 是否在第一张地毯内。
   - 验证结果是否正确。

3. **枚举所有可行解**：
   - 对于 `n = 1`，只有一个地毯，直接判断点是否在该地毯内即可。

#### 第二步：联想类似问题

1. **唤醒知识储备**：
   - 这个问题类似于“覆盖问题”，即判断某个点是否在多个矩形覆盖的区域内。
   - 类似的问题有“矩形覆盖问题”或“二维区间查询问题”。

2. **尝试直接应用**：
   - 直接应用“覆盖问题”的思路，从后往前遍历地毯，判断点是否在当前地毯内。
   - 这种方法在简单实例中是可行的。

3. **改进与适配**：
   - 由于地毯是按照编号从小到大铺设的，后铺设的地毯会覆盖前面的地毯。
   - 因此，可以从最后一张地毯开始检查，直到找到覆盖该点的地毯。
   - 时间复杂度为 `O(n)`，对于 `n <= 10^4` 是可以接受的。

#### 第三步：复杂实例与分解

1. **构造复杂实例**：
   - 构造 `n = 10^4` 的实例，地毯的坐标范围在 `[0, 10^5]`。

2. **分析可分解性**：
   - 这个问题无法分解成更小的子问题，因为每个地毯的铺设都是独立的。

3. **分析可组合性**：
   - 复杂实例的解不能由简单实例的解组合而成，因为每个地毯的铺设顺序是固定的。

#### 第四步：确定解题策略

- **策略选择**：
  - 由于地毯是按照编号从小到大铺设的，后铺设的地毯会覆盖前面的地毯。
  - 因此，可以从最后一张地毯开始检查，直到找到覆盖该点的地毯。
  - 时间复杂度为 `O(n)`，对于 `n <= 10^4` 是可以接受的。

- **具体步骤**：
  1. 读取所有地毯的信息，存储在数组中。
  2. 读取查询点的坐标 `(x, y)`。
  3. 从最后一张地毯开始向前遍历，判断点 `(x, y)` 是否在该地毯内。
  4. 如果找到覆盖该点的地毯，输出其编号；否则输出 `-1`。

通过以上分析，确定使用线性遍历的策略，从后向前检查地毯，直到找到覆盖查询点的地毯。

# P1003 [NOIP 2011 提高组] 铺地毯 - 解题思路

## 第一步：理解问题，从最简单的情形入手

### a. 构造最简单实例
- **最简单情形**：n = 0（没有地毯），或者 n = 1（只有一张地毯）
- 以 n = 1 为例：地毯参数为 (a, b, g, k)，查询点为 (x, y)

### b. 求解与验证最简单实例
- 地毯覆盖的区域为矩形：[a, a+g] × [b, b+k]
- 判断点是否在矩形内：**a ≤ x ≤ a+g 且 b ≤ y ≤ b+k**（边界也算覆盖）
- 若满足条件，输出地毯编号 1；否则输出 -1

### c. 枚举所有可行解
- 本题是查询问题（非最优化），只需逐一检查每张地毯

## 第二步：联想类似问题

### a. 唤醒知识储备
- 与"矩形区域覆盖查询"问题相似
- 类似于图层的概念：后铺的地毯在上层，覆盖下层的
- 本质是：**逆序查找第一个满足条件的元素**

### b. 尝试直接应用
- 由于地毯按编号从小到大铺设，后铺的覆盖先铺的
- **核心策略**：从最后一张地毯开始往前查找，第一个覆盖查询点的地毯就是答案

### c. 改进与适配
- 只需 O(n) 时间复杂度，n ≤ 10^4 完全可接受
- 无需复杂数据结构（线段树等），因为只有一个查询点

## 第三步：复杂实例与分解

### a. 构造复杂实例
- n = 10000 张地毯，查询一个点 (x, y)

### b. 分析可分解性
- 问题无需分解，只有一个查询点，线性扫描即可

### c. 分析可组合性
- 无需组合，找到第一个满足条件的即可

## 第四步：确定解题策略

**算法策略：逆序线性扫描**

**具体步骤：**
1. 读取 n
2. 依次读取 n 张地毯的参数 (a, b, g, k)，存储到数组中
3. 读取查询点坐标 (x, y)
4. **从第 n 张地毯开始向前遍历**（即逆序）：
   - 对于每张地毯，检查条件：a ≤ x ≤ a+g 且 b ≤ y ≤ b+k
   - 如果满足，输出当前地毯编号，算法结束
5. 若遍历完所有地毯都未找到覆盖点，输出 -1

**复杂度分析：**
- 时间复杂度：O(n)，只需一次线性扫描
- 空间复杂度：O(n)，需要存储所有地毯信息（也可边读边处理，但逆序需要先存储）

**关键点：**
- 逆序遍历确保找到的是"最上面"的地毯
- 边界条件要包含"="（边界和顶点也算覆盖）

成员 3: ### **第一步：理解问题，从最简单的情形入手**

#### **a. 构造最简单实例**
最简单的实例是：
- `n = 0`：没有地毯。
- 或 `n = 1`：只有一张地毯。

例如：
- 输入：  
  `1`  
  `0 0 2 2`  
  `1 1`  
  表示一张从 (0,0) 开始、大小为 2×2 的地毯，询问点 (1,1)。

#### **b. 求解与验证最简单实例**
- 对于 `n=1` 的情况：
  - 地毯覆盖区域是：$[a, a+g] \times [b, b+k]$，即 $[0,2] \times [0,2]$。
  - 查询点 (1,1) 显然在该区域内，因此输出地毯编号 `1`。
- 若查询点为 (3,3)，不在任何地毯上，则输出 `-1`。
- 手动验证容易，说明问题本质是：**判断一个点是否落在某个矩形内，并找出最后（最上层）覆盖它的矩形编号**。

#### **c. 枚举所有可行解（非最优化问题）**
这个问题不是最优化问题（如最小值或最大值），而是**查询问题**：找覆盖某点的**最上面一张地毯**（即编号最大的那张，因为后铺的在上面）。
- 所有被覆盖该点的地毯中，取**铺设顺序最晚**（即编号最大）的那个。

---

### **第二步：联想类似问题**

#### **a. 唤醒知识储备**
- 类似问题：
  - **区间覆盖问题**：判断一个点是否被若干区间覆盖。
  - **二维平面点定位**：判断一个点是否在多个矩形中。
  - **事件处理 / 扫描线算法**：用于处理多个几何对象的覆盖关系。
  - **逆序查找**：由于后铺的地毯在上面，可以考虑**从后往前查**。
- 相关数据结构：二维坐标、矩形、点。
- 算法思路：暴力枚举、逆序遍历、几何判断。

#### **b. 尝试直接应用**
- 可以将每张地毯看作一个矩形区域。
- 判断点 $(x,y)$ 是否在矩形 $[a, a+g] \times [b, b+k]$ 内。
- 因为地毯按编号顺序铺设，**后铺的在上层**，所以若多个地毯覆盖该点，**编号最大的那个就是答案**。
- 因此，一个直观方法是：
  - 遍历所有地毯，检查是否覆盖点 $(x,y)$。
  - 记录满足条件的最大编号。

但这是否最优？我们进一步思考。

#### **c. 改进与适配**
- 既然我们关心的是“最上面”的地毯，即**最后铺设但仍覆盖该点的地毯**，
- 那么可以直接**从编号最大的地毯开始，逆序向前遍历**。
- 一旦找到第一个覆盖该点的地毯，就是答案（因为它是最晚铺设的）。
- 这样可以在最坏情况下减少一半操作（平均可能提前终止）。
- 时间复杂度仍为 $O(n)$，但实践中可能更快。

### **第三步：复杂实例与分解**

#### **a. 构造复杂实例**
例如：
- `n = 4`
- 地毯1: (0,0) 到 (2,2)
- 地毯2: (1,1) 到 (3,3)
- 地毯3: (0,3) 到 (2,5)
- 地毯4: (1,2) 到 (4,4)
- 查询点：(2,2)

多个地毯可能覆盖该点，需判断哪个最后铺的还覆盖它。

#### **b. 分析可分解性**
- 输入是一个序列的矩形（按铺设顺序排列）。
- 数据结构是**数组形式的矩形列表**，可以按顺序或逆序访问。
- 无法有效分解为子问题（比如左右分割后不能合并出“最上层”信息），因为覆盖关系依赖全局顺序。
- 因此不适合分治。

#### **c. 分析可组合性**
- 输出只是一个整数（编号或-1）。
- 不具备“子问题解可组合”的特性。
- 例如：前半部分覆盖该点的最大编号，和后半部分的最大编号，取整体最大即可 —— 但这其实就是遍历。
- 所以组合性存在但平凡，无需动态规划。

### **第四步：确定解题策略**

#### ✅ 最佳策略：**逆序遍历 + 几何判断**

核心思路：
1. 地毯按编号从小到大铺设，编号大的在上面。
2. 要找覆盖点 $(x,y)$ 的最上面地毯，等价于找**编号最大**且覆盖该点的地毯。
3. 因此可以从编号 $n$ 到 $1$ 逆序遍历每张地毯：
   - 对每张地毯 $i$，判断点 $(x,y)$ 是否在其范围内。
     - 即判断：  
       $a_i \le x \le a_i + g_i$ 且 $b_i \le y \le b_i + k_i$
   - 一旦找到第一个满足条件的地毯，立即返回其编号。
4. 如果遍历完都没有找到，返回 `-1`。

#### ⚠️ 为什么不用正序遍历？
- 正序遍历也能做：记录所有覆盖的地毯编号，最后取最大值。
- 但逆序遍历的优势是：**一旦命中即可返回，提前终止**，效率更高。

#### 时间复杂度分析：
- 每个地毯做 $O(1)$ 的判断。
- 最坏 $O(n)$，最好 $O(1)$（最后一张就命中）。
- 符合题目 $n \le 10^4$ 的约束，完全可行。

#### 空间复杂度：
- 只需保存所有地毯信息（数组存储），$O(n)$，可接受。

---

### ✅ 总结：解题思路

1. **问题本质**：在按时间顺序叠放的矩形中，查询某点被哪个最上层矩形覆盖。
2. **关键观察**：后铺设的地毯在上层，因此应优先考虑编号大的地毯。
3. **核心策略**：从最后一张地毯开始，逆序向前查找第一个能覆盖查询点的地毯。
4. **几何判断**：对每张地毯，判断点是否在其左下角 $(a,b)$、右上角 $(a+g, b+k)$ 构成的闭区间内。
5. **输出结果**：找到则返回编号，否则返回 `-1`。

该方法简洁、高效、易于实现，适用于全部数据范围。
